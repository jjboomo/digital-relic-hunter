# 尸检报告：wenyan-lang/wenyan 项目

## 一、项目基本信息

- **项目名称**：文言 (Wenyan)
- **GitHub**：wenyan-lang/wenyan
- **语言定位**：古典汉语编程语言

## 二、死因鉴定

### 🚨 直接死因

| 症状 | 诊断 |
|------|------|
| commits 趋于停滞 | 维护者热情耗尽 |
| star 1w+ 但实际使用者寥寥 | 社交媒体效应消散后裸泳 |
| 无企业级应用案例 | 实用性存疑，无正向反馈循环 |
| 文档维护断档 | 单点故障，缺乏社区承接 |

### 🔍 深层病因

**1. 定位悖论**
- 想当「严肃编程语言」→ 语法别扭，无生态
- 想当「极客玩具」→ 无持续新鲜感
- 想当「教育工具」→ 无教学体系和商业闭环
- **结论**：三个方向都没想清楚

**2. 目标用户画像模糊**
```
谁会真的用？
├── 极客？ 用一次发推，then forgotten
├── CS学生？ 考完试就删
├── 汉语言学者？ 发论文够了，不需要生产级
└── 文化爱好者？ 热情峰值太短
```

**3. 护城河为零**
- 竞品多且门槛低
- 本质是个 parser + 关键词映射
- 任何有一定经验的开发者一周可以复现核心功能

**4. 创始人/核心团队的非营利属性**
- 无商业化动机
- 无投资人压力（无投资）
- 无 KPI 驱动
- → 纯靠情怀的不可持续性

**5. 文化借势失败**
- 蹭了「中文编程」热点，但没接住
- 蹭了「文化自信」热点，但没产品化
- 蹭了「AI 叙事」热点，但没提前布局

## 三、秽土转生方案

### 阶段一：战略重定位 —— 从「语言」到「文化操作系统」

**旧定位**：一个编程语言
**新定位**：古典汉语 + 编程 + 教育 + 文化的 IP 矩阵

```
┌─────────────────────────────────────────────────┐
│                   IP 矩阵                        │
├──────────────┬──────────────┬───────────────────┤
│   文言编译器   │  古文益智游戏  │   沉浸式学习 App    │
│  (开源/引流)   │  (C端变现)    │   (B/C混合变现)     │
├──────────────┴──────────────┴───────────────────┤
│              AI 驱动内容生成引擎                   │
│           (底层技术 + 差异化壁垒)                   │
└─────────────────────────────────────────────────┘
```

### 阶段二：技术复活 —— AI-Native 重构

**核心思路**：让 AI 成为「翻译层」，降低使用门槛

```
用户（自然语言/半文言）
      ↓
  [AI 理解层]  ← GPT-4o / Claude / 开源模型微调
      ↓
  [文言语法校验]
      ↓
  [wenyan-lang 编译器]
      ↓
可执行代码 / 学习反馈 / 游戏化成就
```

**关键功能**：

1. **AI 辅助编码**
   - 输入：自然语言需求
   - 输出：自动生成的文言代码
   - 目的：让完全不懂文言的人也能「玩」
   - 壁垒：微调专属模型 + 文言语法数据集

2. **互动式古文编译器**
   - 实时编译 + 错误解释（用古文风格解释 Bug）
   - 类「LEGO」式积木构建

3. **API 化**
   - 开放 `/translate` `/validate` `/compile` API
   - 收费模式：免费 tier + enterprise tier

### 阶段三：内容与社区重启

**Killer Feature**：用代码「复活」古文经典
