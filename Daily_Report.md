## 一、尸检报告（Post-mortem Analysis）

### 1.1 死亡证明

| 指标 | 数据 |
|------|------|
| **最终版本** | 0.12.6 (发布于 2020-03) |

| **Star** | ~13.5k |
| **已知分叉数** | 50+ |
| **QtWebKit 状态** | 已死亡，Qt 已切换至 QtWebEngine (Chromium) |
| **致命伤** | Qt4/Qt5 WebKit 被 Qt 官方放弃，无安全补丁，无现代浏览器兼容 |

### 1.2 死因分析

```
┌─────────────────────────────────────────────────────────────────┐
│                        直接死因                                  │
│   Qt 官方 2016 年宣布放弃 QtWebKit，全面转向 QtWebEngine (CEF)  │
│   → wkhtmltopdf 的技术栈根基被铲断                               │
├─────────────────────────────────────────────────────────────────┤
│                        间接死因                                  │
│   ① WebKit → Chromium 战争失利 (Chrome/Firefox均转向Chromium)   │
│   ② 无商业公司兜底 (项目创始人 @橘和臣 个人维护，无 VC/公司)      │
│   ③ 现代 PDF 生成需求爆炸，但项目未跟进 (CSS3/ES6/Shadow DOM)    │
│   ④ PDF 加密/签名/表单/DRM 等企业功能从未实现                     │
├─────────────────────────────────────────────────────────────────┤
│                        根本死因                                  │
│   "技术债+孤独维护者" 死亡螺旋                                   │
│   QtWebKit 停止更新 → 安全漏洞累积 → 贡献者流失 → 影响力下降     │
│   → 维护者放弃 → 项目死亡                                        │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 尸检发现

**优势遗产（仍有价值的部分）：**
- ✅ 命令行工具设计哲学优秀（简单、Unix风格）
- ✅ 零依赖单文件分发（Windows/macOS/Linux开箱即用）
- ✅ 积累了大量生产环境使用案例和踩坑文档
- ✅ 品牌知名度：在 "HTML to PDF" 领域几乎等于代名词

**腐烂部分（必须丢弃的遗产）：**
- ❌ QtWebKit 渲染引擎 → 无法渲染现代 Web 内容
- ❌ Qt4/Qt5 依赖 → 与现代系统不兼容
- ❌ SSL/TLS 实现 → 存在已知漏洞
- ❌ 对 HTTP/2, gRPC, WebSocket 等现代协议的支持

**竞争者现状：**

| 工具 | 引擎 | 维护状态 | 市场份额估计 |
|------|------|----------|------------|
| Puppeteer/Playwright | Chromium | ✅ 活跃 | 60%+ |
| weasyprint | WeasyPrint | ✅ 活跃 | 15% |
| pdfkit | wkhtmltopdf | ⚠️ 勉强 | 10% |
| princexml | 自研 | ✅ 商业 | 5% |
| **wkhtmltopdf** | **QtWebKit** | **❌ 死亡** | **~5%** |

### 2.2 方案A：Chromium 完全重写（推荐长期战略）

```python
# 架构设计
import asyncio
from playwright.async_api import async_playwright
import subprocess

class WkhtmltopdfReborn:
    """
    新wkhtmltopdf: Playwright(Chromium) + 兼容层
    保留原有CLI接口，底层替换为现代引擎
    """
    
    def __init__(self, engine='chromium'):
        self.engine = engine
        
    async def convert(self, source, output, options):
        async with async_playwright() as p:
            browser = await p.chromium.launch()
            page = await browser.new_page()
            
            # 支持URL或HTML字符串
            if source.startswith('http'):
                await page.goto(source, wait_until='networkidle')
            else:
                await page.set_content(source)
            
            # 核心转换
            await page.pdf(
                path=output,
                format=options.get('page-size', 'A4'),
                margin=options.get('margin', {}),
                print_background=options.get('background', True),
                display_header_footer=options.get('header', False),
                # 新增现代功能
                **self._map_options(options)
            )
            await browser.close()
            
    def _map_options(self, options):
        """兼容原有CLI参数映射"""
        return {
            'landscape': options.get('orientation') == 'Landscape',
            'scale': options.get('zoom', 1.0),
            'prefer_css_page_size': options.get('enable-local-file-access', False),
        }
```

**关键迁移策略：**

```bash
# 保持原有CLI接口，用户无感知迁移
# 旧: wkhtmltopdf http://example.com output.pdf
# 新: wkhtmltopdf-reborn http://example.com output.pdf

# 提供兼容层脚本
#!/bin/bash
# wkhtmltopdf -> 自动路由到 reborn 版本
exec wkhtmltopdf-reborn "$@"
```
